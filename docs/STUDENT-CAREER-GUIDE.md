## üéì What You're Actually Learning (And Why It Matters)

### Lab 1: Threat Modeling & Secure Prompting

**The Skill:** Understanding how attackers think about LLMs

**Why It Matters:** Before you can defend, you must attack. Lab 1 puts you in the attacker's seat using real jailbreak tools (Gandalf, RedArena). The threat model you produce is exactly what security teams present to executives before launching AI products.

**Career Translation:**
> *"In my previous work, I developed threat models for LLM applications using OWASP LLM Top-10 and MITRE ATLAS frameworks. I can show you documented attack scenarios and the controls I recommended."*

### Lab 2: Secure Code Review with LLMs

**The Skill:** Making AI assistants useful for security without hallucinating false positives

**Why It Matters:** Every company has thousands of unreviewed security alerts. You'll learn to use LLMs to triage them‚Äîand more importantly, to measure when the AI is helping vs. hurting.

**Career Translation:**
> *"I built evaluation pipelines using promptfoo that measure precision and recall for LLM-assisted code review. Here's my metrics CSV showing a 23% improvement in finding real vulnerabilities with 40% fewer false positives."*

### Lab 3: Scanner Integration (Checkov + Semgrep + LLM)

**The Skill:** Combining traditional security tools with AI intelligence

**Why It Matters:** Static analysis tools produce overwhelming output. You'll learn to use LLMs as a "first-level analyst" who summarizes what actually matters.

**Career Translation:**
> *"I've integrated Checkov and Semgrep into LLM triage workflows. I can show you before/after evidence where AI reduced alert fatigue by prioritizing the 7 critical findings from 200+ raw scanner outputs."*

### Lab 4: Guardrails & Red Team

**The Skill:** Building and validating defensive controls

**Why It Matters:** This is the money skill. You'll implement input/output filters, measure block rates, and prove with data that your guardrails actually work.

**Career Translation:**
> *"I designed and implemented guardrail policies that achieved a 94% block rate against prompt injection attacks. Here's my unguarded vs. guarded comparison showing the before/after metrics."*

### Final Project: Secure RAG or Safe Agent

**The Skill:** End-to-end secure system implementation

**Why It Matters:** You'll build something that could actually ship. RAG systems that cite sources, or agents that stay within allowed tools‚Äîwith full evaluation and logging.

**Career Translation:**
> *"I designed and implemented a secure RAG system with citation requirements and guardrails. It passed automated red-team evaluation with 95%+ JSON validity, 87% safety rate, and 82% citation accuracy. Here's my GitHub repo with full documentation."*
